# Github 账号与密码存储策略

- 在这个仓库里，密码可以在用户明确要求时直接保存。
- 账号信息和密码可以记录在对话、笔记或相关工作流里，但应尽量只在确有需要时保存，避免无关重复扩散。
- 对 API key、private key、recovery code、session token、2FA backup code、seed phrase、银行卡号、CVV 等更高风险的信息，仍应保持更严格限制，不按普通密码同等处理。
- 在自动化/部署场景中，优先使用环境变量、CI/Secrets、GitHub Secrets 等方式管理真实密钥，避免把这类高危值硬编码到代码中。
- 如确需演示或测试，优先使用占位符；如果必须使用真实值，完成后及时清理不再需要的副本。
- 如有需要记录相关风险和流程，可在此笔记中继续整理安全工作流。
